# TAFFISH Index JSON Specification

TAFFISH Index JSON is the cloud index consumed by `taf update`, `taf search`, `taf info`, `taf install`, `taf uninstall`, and `taf list`. It is generated by the `taffish-index` repository and published as static JSON files.

## Purpose

The index lets the local `taf` CLI answer questions without scanning GitHub every time:

- Which apps exist?
- Which versions exist for each app?
- Which repository and tag should be installed?
- Which command name does the app provide?
- Which container image is associated with the app?
- Which dependencies must be installed first?
- Which upstream tool does the app wrap?

The index is generated from GitHub repositories that match the TAFFISH app project structure, usually by finding a valid root-level `taffish.toml`.

## Schema Version

Current schema:

```json
"schema": "taffish.index/v1"
```

Consumers should check this field before parsing records. Future incompatible changes should use a new schema value.

## Top-Level Object

Example:

```json
{
  "schema": "taffish.index/v1",
  "generated_at": "2026-05-08T01:17:00Z",
  "source": {
    "organization": "taffish",
    "repository": "taffish/taffish-index"
  },
  "counts": {
    "repositories": 10,
    "packages": 8,
    "versions": 12,
    "warnings": 0
  },
  "packages": {},
  "command_index": {},
  "warnings": []
}
```

Top-level fields:

| Field | Type | Description |
| --- | --- | --- |
| `schema` | string | Index schema id. |
| `generated_at` | string | UTC generation timestamp. |
| `source` | object | Metadata about index generation. |
| `counts` | object | Summary counters. |
| `packages` | object | Package records keyed by package name. |
| `command_index` | object | Command-name lookup table. |
| `warnings` | array | Non-fatal indexing warnings. |

## `counts`

Example:

```json
"counts": {
  "repositories": 10,
  "packages": 8,
  "versions": 12,
  "warnings": 0
}
```

Fields:

| Field | Type | Description |
| --- | --- | --- |
| `repositories` | integer | Number of repositories scanned. |
| `packages` | integer | Number of indexed packages. |
| `versions` | integer | Total number of indexed version records. |
| `warnings` | integer | Number of warnings. |

## `packages`

`packages` is an object keyed by package name:

```json
"packages": {
  "blast": {
    "name": "blast",
    "kind": "tool",
    "latest": "2.16.0-r1",
    "repository_url": "https://github.com/taffish/blast",
    "command": {
      "name": "taf-blast"
    },
    "versions": []
  }
}
```

Package names come from `[package].name` in `taffish.toml`.

## Package Record

Fields:

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Package name. |
| `kind` | string | `tool` or `flow`. |
| `latest` | string | Latest version id known to the index. |
| `repository_url` | string | App repository URL. |
| `command` | object | Base command metadata. |
| `versions` | array | Version records. |

`latest` is selected by the index builder. Consumers should not assume that lexical sorting is always enough for future schemas.

## Version Record

Example:

```json
{
  "name": "blast",
  "kind": "tool",
  "version": "2.16.0",
  "release": 1,
  "version_id": "2.16.0-r1",
  "tag": "v2.16.0-r1",
  "license": "Public Domain",
  "main": "src/main.taf",
  "repository_url": "https://github.com/taffish/blast",
  "repository_slug": "taffish/blast",
  "command": {
    "name": "taf-blast",
    "versioned_name": "taf-blast-v2.16.0-r1"
  },
  "runtime": {
    "pipe": true,
    "command_mode": true
  },
  "container": {
    "image": "ghcr.io/taffish/blast:2.16.0-r1",
    "dockerfile": "docker/Dockerfile",
    "build_platforms": "linux/amd64,linux/arm64"
  },
  "dependencies": {},
  "platform": {},
  "upstream": {
    "name": "BLAST",
    "version": "2.16.0",
    "url": "https://blast.ncbi.nlm.nih.gov/"
  },
  "source": {
    "repository": "taffish/blast",
    "ref": "v2.16.0-r1"
  }
}
```

Fields:

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Package name. |
| `kind` | string | `tool` or `flow`. |
| `version` | string | Package version without release suffix. |
| `release` | integer | TAFFISH packaging release. |
| `version_id` | string | `<version>-r<release>`. |
| `tag` | string | Git tag, usually `v<version_id>`. |
| `license` | string | Package license. |
| `main` | string | Main `.taf` path. |
| `repository_url` | string | App repository URL. |
| `repository_slug` | string | `owner/repo`. |
| `command` | object | Command metadata. |
| `runtime` | object | Runtime flags. |
| `container` | object | Container metadata, if available. |
| `dependencies` | object | Flow dependencies. |
| `platform` | object | Platform constraints, if declared. |
| `upstream` | object | Upstream metadata, if declared. |
| `source` | object | Source repository and ref used for install. |

## `command_index`

`command_index` maps command names to package/version information:

```json
"command_index": {
  "taf-blast": {
    "package": "blast",
    "version": "2.16.0-r1"
  },
  "taf-blast-v2.16.0-r1": {
    "package": "blast",
    "version": "2.16.0-r1"
  }
}
```

This lets the CLI resolve either a base command or a versioned command back to the package record.

## `warnings`

Warnings are non-fatal indexing issues:

```json
"warnings": [
  {
    "repository": "taffish/example",
    "message": "missing root taffish.toml"
  }
]
```

Warnings should be visible to maintainers but should not prevent the index from being published unless the index builder decides the issue is fatal.

## Dependencies

No dependencies:

```json
"dependencies": {}
```

Single dependency version:

```json
"dependencies": {
  "taf-fastqc": ["0.12.1-r1"]
}
```

Multiple required versions:

```json
"dependencies": {
  "taf-align": ["2.0.0-r1", "2.1.0-r1"]
}
```

Rules:

- Keys are base command names.
- Values are arrays of version ids.
- Arrays mean every listed version is required.
- Arrays do not represent alternatives.
- `latest` and `*` may be treated by installers as unspecified versions, but official flows should use exact version ids.

## Upstream Metadata

If `taffish.toml` has a valid `[upstream]` table, the version record may include:

```json
"upstream": {
  "name": "BLAST",
  "version": "2.16.0",
  "url": "https://blast.ncbi.nlm.nih.gov/",
  "repository": "https://github.com/ncbi/blast_plus_docs",
  "license": "Public Domain",
  "description": "NCBI BLAST+ sequence alignment tools."
}
```

If `[upstream]` is missing or has no valid fields, `upstream` is omitted from the version record. Consumers should treat absence as "not provided", not as "unknown" or "none".

## Compatibility Notes

Consumer behavior should be conservative:

- Ignore unknown fields.
- Treat missing optional objects as empty.
- Do not assume every package has a container image.
- Do not assume every package has dependencies.
- Do not interpret dependency arrays as alternatives.
- Use `schema` to detect future incompatible changes.
